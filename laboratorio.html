Requisitos Funcionales

Los requisitos funcionales describen lo que el sistema debe hacer, es decir, sus funciones y comportamientos.

Simulación: Identifica Requisitos Funcionales

Para un sistema de gestión de pedidos en línea, selecciona las afirmaciones que representan requisitos funcionales.

Ejercicio: Escribe un Requisito Funcional

Considerando un sistema para una aplicación de reservas de citas médicas, escribe un requisito funcional para la cancelación de citas. Un buen requisito funcional debe ser claro, conciso y describir una acción del sistema.

Requisitos No Funcionales

Los requisitos no funcionales describen cómo el sistema debe funcionar, es decir, sus cualidades y limitaciones.

Simulación: Clasifica Requisitos No Funcionales

Arrastra cada requisito no funcional a su categoría correcta.

El sistema debe soportar 1000 usuarios concurrentes.
Los datos del usuario deben ser encriptados en tránsito.
La navegación del sitio web debe ser intuitiva.
El sistema debe estar disponible el 99.9% del tiempo.
Rendimiento (ej. velocidad, capacidad de carga)
Seguridad (ej. protección de datos, autenticación)
Usabilidad (ej. facilidad de uso, interfaz)
Fiabilidad/Disponibilidad (ej. tiempo de actividad, recuperación)
