Rol: 1 estudiante actúa como cliente; los demás son el equipo de desarrollo.

Objetivo

Identificar y diferenciar requisitos funcionales (lo que el sistema/avión debe hacer) y requisitos no funcionales (cómo debe ser o comportarse), a través del diseño y construcción de un avión de papel según especificaciones del cliente.

Instrucciones

  1. Asignar roles: el grupo elige a una persona como cliente; los demás son el equipo de desarrollo.
  2. Levantar requisitos no funcionales (características): el equipo entrevista al cliente y registra características del avión (tamaño, color, materiales, estilo, peso estimado, etc.).
  3. Levantar requisitos funcionales (comportamiento): el cliente explica lo que el avión debe lograr (distancia de vuelo, estabilidad, giros, planeo lento, acrobacias, etc.).
  4. Construcción: el equipo fabrica el avión de papel respetando los requisitos acordados.
  5. Presentación y prueba: el grupo expone cuáles requisitos cumplieron y realiza un vuelo de prueba.

Presentación y prueba (Validación y Verificación)

En esta fase, el grupo actúa como si estuviera entregando el producto final a un cliente real. El objetivo es validar que el avión cumple lo que el cliente pidió y verificar que las especificaciones se cumplan en la práctica.

Presentación (Validación)

  • Mostrar el avión terminado al cliente y a la clase.
  • Explicar los requisitos funcionales y no funcionales definidos.
  • Indicar cómo se implementó cada requisito.
  • Justificar cambios o ajustes realizados durante la construcción.

Prueba (Verificación)

  • Realizar vuelos de prueba del avión de papel.
  • Medir la distancia de vuelo (en metros).
  • Medir el tiempo de planeo (en segundos).
  • Observar y registrar si realiza giros, acrobacias o se mantiene estable según lo solicitado.
  • Comparar resultados con los requisitos definidos para confirmar si se cumplieron.
